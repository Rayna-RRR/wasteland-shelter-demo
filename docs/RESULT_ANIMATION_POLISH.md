# 结果动效打磨

## 动效位置

- 招募页：幸存者结果卡出现时增加短暂的淡入、上移和轻微缩放动效。
- 招募页：SSR 结果额外增加一次克制的橙色光晕，不影响普通结果展示。
- 值勤页：值勤结果卡出现时增加短暂的淡入、上移和轻微缩放动效。
- 值勤页：资源变化、疲劳变化、健康变化的结果小卡增加一次轻量高亮。

## 为什么选择这两处

- 招募结果是玩家获得新幸存者的核心反馈点，适合用入口动效强化“信号锁定”的感觉。
- 值勤结果是玩家派遣后的后果反馈点，适合用结果卡和变化数值的短高亮强化“行动有代价”的感觉。
- 这两处都只发生在结果数据更新后，不需要改动后端结算、数据库或玩法系统。

## 使用的动效样式

- 使用 WXSS `@keyframes`，不依赖额外库。
- 结果卡使用 `opacity`、`translateY`、`scale` 做 320ms 到 430ms 的入口动效。
- 值勤变化小卡只使用 `box-shadow` 做短高亮，避免文字跳动或遮挡。
- JS 只在结果更新时切换 `resultAnimationIndex`，让重复招募或重复值勤时可以重新触发动效。

## 控制范围

- 整页、按钮、列表不加入连续动效，避免页面显得浮躁。
- 本轮不加入音效、粒子、图片或复杂连段动画。
- 业务逻辑不依赖动画状态；动画失败时结果仍然正常显示。
- 值勤派遣接口参数保持不变。

## 手动测试清单

1. 打开招募页，执行一次招募，确认结果卡短暂淡入并上移进入。
2. 连续执行多次招募，确认新结果仍会触发入口动效。
3. 如抽到 SSR，确认结果卡只有轻微高亮，不遮挡姓名、稀有度和角色信息。
4. 打开值勤页，选择幸存者并派遣任意值勤，确认结果卡短暂出现动效。
5. 检查资源变化、疲劳、健康小卡的正负变化颜色仍然清晰可读。
6. 连续派遣或休整，确认结果卡会再次触发动效。
7. 在缺少头像、特质或状态标签的数据下确认页面不崩溃，仍使用已有兜底展示。
